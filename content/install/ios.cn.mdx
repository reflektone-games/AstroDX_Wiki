### 安装谱面
- 新版 AstroDX 更新使安装新谱面的过程变得更简单但也更复杂。想必待会你就会明白我的意思。

**在本指南中，`合集` 基本上就是谱面的文件夹。你应该熟悉这种结构：** `24. PRiSM/Chronomia`

<Accordions>

如果你想更改合集详情，比如说向合集中添加新的单个谱面，那就有点复杂了。

<Callout title="你想进一步修改你的合集吗？">
  如果你对修改不感兴趣，**跳过下一节**（甚至可以退出本指南，因为你已经成了）。
</Callout>
在你的 `AstroDX` 文件夹中，你会找到一个 `collections` 文件夹和一个 `levels`
文件夹。- `collections` 文件夹仅存储有关各个合集的信息，位于子文件夹中。- `levels` 文件夹仅存储与谱面相关的文件。

如果你想修改合集，比如向其中添加一个谱面，你需要进入该合集的文件夹并更改 `manifest.json` 文件中的信息。

你可以在[这里](https://astrodx.notion.site/Collection-Manifest-Details-8f966de6971748e0ac01c0a0a6054594)阅读关于该文件的信息，但**简而言之**：

在 `manifest.json` 文件中，你会看到这种格式。阅读这个应该能让你了解它是如何工作的。

```json
{
  "name": "合集名称",
  "id": null, // 你可以忽略这个
  "serverUrl": null, // 你可以忽略这个
  "levelIds": ["这里的每一项", "都是levels文件夹中", "的一个文件夹"]
}
```

这就是 `manifest.json` 文件的工作原理。

#### LocalSend – AirDrop 的开源替代方案

如果你想从 PC 传输文件到 iPhone/iPad（除非是 Mac，那就直接用 AirDrop），这一节是必不可少的。

PC 和移动设备上都需要下载 LocalSend。

<Cards>
  <Card
    title="App Store"
    href="https://apps.apple.com/cn/app/localsend/id1661733229"
  ></Card>
  <Card
    title="PC 端下载官方网站"
    href="https://localsend.org/"
  ></Card>
</Cards>
确保两台设备都连接到同一个 WiFi 网络。

在移动设备上保持 LocalSend 应用打开。在 PC 上：

<Steps>
  <Step> 打开发送菜单。 </Step>
  <Step>
    {" "}
    选择"文件"，并选择包含谱面的 `.adx` 文件。{" "}
  </Step>
  <Step>
    {" "}
    你的 iPhone 或 iPad 应该会显示在此菜单中，带有一个代号。点击该名称。{" "}
  </Step>
  <Step>
    {" "}

  <Step>

# 常见问题解答

<Accordions>
  <Accordion title="我已经把 .rar/.zip 文件放入了 `/levels` 文件夹，但还是跑不出曲目！">
    对于单个谱面，你需要先解压。

    不过，使用[.adx 归档方法](#安装谱面)可能会更好。
  </Accordion>

  <Accordion title="有没有办法在玩音游时关闭手机通知？我总是不小心滑到。">
    使用[**引导访问**](https://www.google.com/search?q=ios+guided+access)功能。
  </Accordion>

  <Accordion title="关于文件兼容性的信息">
    这个问题有点难表述，所以我就直接在这里放上相关信息了。

    AstroDX 在 iOS 上官方支持的编解码器类型：
      - **视频：** H.264
      - **音频：** OGG Vorbis（扩展名为 .ogg）**或** mp3（扩展名为 .mp3）
  </Accordion>

  <Accordion title="我已经把 mv.mp4（或 bg.mp4）文件放入了文件夹，但视频没在播放！">
    AstroDX 只检测 `pv.mp4` 作为视频文件。顺带一提，`mv.mp4` 是在 [Majdata](https://rentry.org/maiguide) 中运行视频所需的文件名。
  </Accordion>

  <Accordion title="为什么 AstroDX 不在 App Store 上？">
    原因有两方面。我们目前还不符合在 App Store 上发布的条件，而且我们希望在这样做之前先成为一个成熟的游戏。
  </Accordion>

  <Accordion title="可以发布 .ipa 文件让我们自行侧载吗？">
    我们也希望可以，但这是我们不能这样做的原因：

    > 目前不推荐 ipa 侧载。简而言之，发布 ipa 文件会让人们能够在 App Store 上以自己的名义分发 AstroDX，并可能通过插入广告或添加付费限制从中获利。我们希望确保游戏能正当地发布到 App Store，所以我们不打算发布任何 ipa 文件。

    **长话短说：** 一颗老鼠屎，坏了一锅粥。
  </Accordion>

  <Accordion title="为什么 TestFlight 经常满员？我什么时候才能玩到？">
    简短的回答是：太多玩家想玩这个游戏。

    之所以有特定数量的名额，是因为我们目前只能负担这么多。为你们开放这些组需要成本，而我们是自掏腰包。
  </Accordion>
  
  <Accordion title="如果名额这么快就用完了，为什么不多开几个组？为什么不开 E 组、F 组等等？">
    请看上面的回答。
  </Accordion>

  <Accordion title="[2.0 Beta] 为什么我们要改变歌曲存储布局？">
    在 AstroDX 的早期版本中，曲目可以放在顶层（直接放在 `levels` 文件夹内），或者你可以将一个或多个曲目放在另一个命名文件夹中以创建**合集**。这让在文件系统中直接自定义合集变得方便，尽管它也引入了不少问题。

    **不能在多个合集中使用相同的曲目**。由于合集以文件夹的形式存在，很难找到一种方法将相同的曲目同时放入多个文件夹。即使可行，在删除曲目或将曲目移至其他位置时，管理快捷方式和链接肯定会很困难。

    **多个曲目可以有相同的名称**。尽管我们使用文件夹来分组曲目，但在游戏中这些曲目会展开为一个单一列表。这使得可能有两个或更多同名的文件夹，在游戏的曲目字典中造成键冲突。当你将曲目添加到众多文件夹中时，很难识别是否已存在同名的文件夹。

    通过使用表示合集的替代方式，尽管我们使编辑合集数据变得稍微复杂了一些（我们正在努力添加游戏内合集管理器来解决这个问题），但我们能够解决上述问题了。
  </Accordion>
</Accordions>
